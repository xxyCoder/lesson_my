- 引用式赋值考点
    1. 复杂数据类型在内存中存在哪里？
        栈内存   简单数据  +  函数栈
        堆内存  复杂数据     {name: '龙', age: 17}   &0x1123 -> 变量 
    2.  let a = b =  {name: 'x', age: 17}     a   b   不是简单数据类型的值的拷贝， 
        a b 指向同一个地址  
    3. 将对象 传递给函数的时候， 引用式赋值  不太好的写法， 因为， 在函数内部，
        有可能会修改这个对象 导致理解错误 
    4. person 重新赋值   指向的地址就改变了
